# 🧭 당신의영양제 — 소비자 만족·실용성 기획 (방향 설정)

> 작성: 2026-06-29 · 목적: "한 번 추천받는 사이트" → **"매일 챙기게 되는 건강 비서"**
> 기준: 소비자 만족(인지부하↓·신뢰·보살핌 느낌) + 실용성(매일 쓸 이유) + 재방문(습관 루프)

---

## 1. 리서치 요약 (근거)

- **건강·웰니스 앱 리텐션은 매우 낮다**: 30일 잔존 ≈ 7.9%, 3개월 내 90%+ 이탈.
  → **신뢰(1회성 가치)만으로는 다시 안 온다.** 매일 쓸 이유 = 습관 루프가 필요.
- **끈끈함의 동력**(2025): 강한 온보딩 · **개인화(단계별 여정)** · **스마트 알림(90일 내 1회 푸시 → 잔존 3배)** · 습관화(스트릭/체크) · **"보살핌받는 느낌(feel seen)"**.
- **한국 승자 패턴(필리·필리케어 등)**: `추천(근거) → 매일 복용 기록 → 알림 → 재구매/정기구독`.
  - 필리케어: 섭취 **기록 + 무료 알림 + 건기식 정보 + 커뮤니티**, 100만+ 추천.
  - 후기 핵심: *"처음으로 영양제 통을 비웠다 / 알림 오기 전에 챙겨 먹게 됐다"* = **습관화가 만족의 본질.**

출처: ScaleWithFuture(리텐션), GetStream(2026 벤치마크), Pilly·PillyCare(국내 앱).

## 2. 현재 상태 진단

### 강점 (이미 구축)
- ✅ 근거 기반 추천 엔진(점수·시너지·길항·약물경고) + 설문 연결
- ✅ 성분별 **근거 코퍼스**(17종, NIH ODS·NCCIH 등 출처 인용) 화면 노출
- ✅ 복용 기간 가이드(🟢지속/🟡점검/🔴주기), 함량당 최저가, 지도(약국·응급실)
- ✅ 내추럴 그린 앱 디자인 + Pretendard

### 만족·실용성 갭 (개선 대상)
| # | 갭 | 영향 |
|---|----|------|
| G1 | **습관 루프 부재** — 웹 "내 영양제 등록/알람"이 죽은 버튼(미작동·미저장) | 재방문 동기 0 → 리텐션 절벽 |
| G2 | **개인화 narrative 약함** — "왜 *나에게* 이것"이 약함. 설문의 lifestyle(흡연·음주·운동) 미반영 | "feel seen" 부족 |
| G3 | **결과 인지부하** — 카드가 길고 정보 과다. 핵심 요약·우선순위 부재 | 첫인상 만족↓ |
| G4 | **재평가·진척 루프 없음** — 🟡점검/🔴주기 안내만 있고 실제 리마인드·후속 없음 | 안전 가치가 1회성에 그침 |
| G5 | **재구매 실용성 약함** — 가격비교는 있으나 "떨어질 때쯤" 리마인드 없음 | 지속 사용·전환 손실 |

## 3. 방향 (제품 원칙)

1. **신뢰는 이미 자산 — 이제 "매일의 습관"으로 연결한다.** (추천 → 기록 → 알림 → 재평가)
2. **덜 보여주고 더 명확히.** 결과는 "핵심 3가지 + 안전 요약" 먼저, 상세는 펼치기.
3. **나를 위한 설명.** 선택한 고민·생활습관을 문장으로 되돌려준다("눈 피로 + 야간운전 → 루테인").
4. **키 없이도 동작.** localStorage·브라우저 알림 우선 → 키/계정은 나중(현재 인프라 철학과 일치).
5. **의료 경계 준수.** 진단·치료 아님, 정보 제공. 안전(병용·상한)은 강하게, 효능은 절제해서.

## 4. 우선순위 로드맵 (만족·실용성 기준)

```
P1  내 루틴(My Routine) — 복용 기록 + 리마인더        ← 습관 루프, 최대 레버
P2  결과 만족도 — 핵심 요약 + 개인화 narrative          ← 첫인상·feel seen
P3  재평가·진척 루프 — 점검/주기 리마인드 + 컨디션 체크인  ← 안전 가치 지속화
P4  재구매 실용성 — 소진 예측 리마인드                  ← 지속 사용·전환
P5  (기존 기획) DUR 병용금기 확장 · 낱알식별            ← 신뢰 심화(2차)
```

### P1. 내 루틴 — 복용 기록 + 리마인더 〔핵심〕
- 추천 결과/검색 → **"내 영양제에 추가"** → `/my` 대시보드(웹 신규 라우트).
- **오늘 먹을 것 체크**(엔진 스케줄 아침/저녁 활용) + **연속 복용일(streak)**.
- 저장: `localStorage`(키 없이 동작) → 이후 계정·서버 동기화.
- 알림: 브라우저 알림(무료) → 카카오 알림톡(키 있을 때, 기존 server/api/kakao.js 연계).
- 모바일 앱(my-supplements.jsx·alarm.jsx)에 이미 개념 존재 → **웹에 이식·정합**.
- 성공지표: 추가율, 7일 내 체크 1회 이상(=습관 진입), 알림 동의율.

### P2. 결과 만족도 — 핵심 요약 + 개인화 narrative
- 결과 상단 **"OO님 핵심 3가지"** 요약 카드: 상위 3성분 + 각 1줄 이유(선택 고민 매핑).
- **안전 요약 배너**: 피해야 할 조합/복용약 경고를 맨 위로(불안 먼저 해소).
- 생활습관 반영 코멘트(흡연→비타민C 가산, 음주→간 코멘트 등 — 엔진/문구 레벨).
- 상세 카드는 "펼치기"로 접어 인지부하↓.

### P3. 재평가·진척 루프
- 🟡점검(90일)/🔴주기(8주) → 실제 **"n주 후 점검 알림"** 예약.
- 1탭 **컨디션 체크인**(피로/수면/장 등 주관 5점) → 간단 추세 그래프.
- 재설문 유도("3개월 됐어요, 다시 볼까요?").

### P4. 재구매 실용성
- 등록 시 수량·1일 복용량 → **소진 예정일 추정** → "곧 떨어져요" 리마인드 + 최저가 재노출.

### P5. (기존 기획) 신뢰 심화
- DUR(식약처) 병용금기 확장, 낱알식별 — 기존 IDEAS_medical_services.md 로드맵 계승.

## 5. 추천 시작점

**P1(내 루틴) 먼저.** 이유: ① 리텐션 절벽을 직접 공략하는 최대 레버, ② 모바일에 개념이 있어 웹 이식이 빠름, ③ 키 없이 localStorage+브라우저 알림으로 즉시 동작, ④ 이후 P2~P4가 모두 "내 루틴" 위에 얹힘.
