# 🏥 확장 서비스 브레인스토밍 — 의료 정보·지도

"당신의 영양제"를 **건강 종합 비서**로 키우기 위한 추가 서비스 연구.
핵심 기준: **실제 공공 데이터/API가 있는가** + **기존 앱과 시너지** + **난이도**.

> 좋은 소식: 한국은 약국·응급실·의약품 데이터를 정부가 공개(공공데이터포털 data.go.kr)해서,
> 아래 대부분이 **API만 연결하면 바로 구현 가능**합니다. (네이버쇼핑 연결한 것과 같은 방식)

---

## 후보 서비스 모듈

### 🥇 A. 지금 문 연 약국·병원 찾기 (지도) — 최우선 추천
처음에 구상했던 "병원·약국 정보 공유"가 바로 이거예요. 대중 수요가 가장 크고 데이터도 준비돼 있어요.

| 기능 | 설명 | 데이터 |
|------|------|--------|
| 24시·심야·공휴일 약국 | 지금/오늘 문 연 약국을 지도에 | 국립중앙의료원 **전국 약국 정보** API (영업요일·시간 포함) |
| 응급실 실시간 | 응급실 위치 + **실시간 가용 병상** | **전국 응급의료기관 정보** API (실시간 병상) |
| 야간 소아진료 | 달빛어린이병원(밤에 여는 소아과) | E-Gen 응급의료포털 데이터 |
| 심장충격기(AED) | 가까운 AED 위치 | 전국 **AED 위치** API |

- **지도**: 카카오맵 / 네이버지도 API로 표시 (무료 한도 충분)
- **난이도**: 중 (API + 지도 SDK). 키 발급은 네이버쇼핑처럼 무료
- **시너지**: "영양제 사러 갈 약국" → "문 연 약국 찾기"로 자연 연결

### 🥈 B. 약-영양제 병용금기 체크 (DUR) — 시너지 최고
우리가 이미 만든 **상호작용 엔진을 "진짜 의약품"까지 확장**. 차별화의 핵심.

- 식약처 **DUR(의약품안전사용서비스) API**: 병용금기·임부금기·노인주의·효능군중복·투여기간주의 등
- 사용자가 "내 약" 등록 → 추천 영양제와 **실제 병용금기 자동 경고**
- 지금은 와파린·스타틴 등 몇 개만 하드코딩 → DUR 연결 시 **수천 개 의약품 자동 커버**
- **난이도**: 중 / **시너지**: ★★★ (기존 엔진 그대로 확장)

### 🥉 C. 내 약 알아보기 — 낱알 식별
- 식약처 **의약품 낱알식별 API**: 알약 모양·색·각인으로 약 이름 찾기
- "이 알약 뭐지?" → 검색 → 약 정보 + 우리 DUR 경고로 연결
- "내가 먹는 약 등록"을 **사진/각인으로 쉽게** 만들어줌
- **난이도**: 중상 (이미지/각인 검색 UI) / **시너지**: ★★

### 그 외 후보 (2차)
| 모듈 | 설명 | 데이터 |
|------|------|--------|
| 증상 → 진료과 추천 | "이런 증상은 무슨 과?" | 자체 룰 + 병원 검색 |
| 예방접종·건강검진 알림 | 시기 알림(독감철 등) | 질병청 가이드 |
| 복약 캘린더 통합 | 영양제+처방약 한 번에 알람 | 기존 알람 확장 |
| 의약품 정보 검색 | 효능·용법·부작용 | 식약처 의약품 정보 API |

## 🎯 추천 로드맵 (우선순위)

```
1단계  B. DUR 병용금기  ← 기존 엔진 확장, 시너지 최고, 지도 불필요
2단계  A. 문 연 약국·응급실 지도  ← 대중 수요 1위, 원래 아이디어
3단계  C. 낱알 식별 + 내 약 등록 고도화
4단계  증상→진료과, 건강검진 알림 등
```

**왜 이 순서?**
- **B(DUR)** 는 이미 있는 상호작용 엔진에 데이터만 붙이면 돼서 **가성비 최고** + "당신의 영양제" 정체성(안전·근거) 강화.
- **A(지도)** 는 새 화면이 필요하지만 **사용자 유입·재방문**이 큰 킬러 기능. 처음 구상과도 일치.

## 🧭 통합 비전

```
        당신의 영양제  →  당신의 건강 비서
  ─────────────────────────────────────────
  💊 영양제 추천(근거기반)   ← 지금 완성
  ⚠️ 약·영양제 병용 안전     ← B (DUR)
  🗺️ 문 연 약국·응급실       ← A (지도)
  📋 내 약·영양제 통합 관리   ← 기존 + C
  ⏰ 복약 알람(카톡/푸시)     ← 지금 완성
```
하나의 앱에서 "뭘 먹을지 → 안전한지 → 어디서 살지/받을지 → 언제 먹을지"가 끊김 없이 이어짐.

## ⚠️ 유의점
- 응급실 **실시간 병상**은 생명과 직결 → "참고용, 전화 확인 권장" 고지 필수
- 의료법: 진단·치료가 아닌 **정보 제공**으로 일관되게 포지셔닝
- 지도/공공 API는 **이용약관·표시의무**(출처 명시) 준수
