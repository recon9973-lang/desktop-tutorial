# AI 콘텐츠 작성 스킬 (GEO/AEO · E-E-A-T 기반)

기존 키워드 반복형 SEO를 탈피해, **AI 모델이 고평가하는 맥락(Context)·전문성(Expertise)·데이터 구조화**에
초점을 맞춘 심층 콘텐츠를 만든다. 근거: `AI_Content_Creation_Guide.md`, 구현 참고: `lib/post-generator.js`.

## 언제 쓰나
- 블로그/칼럼/랜딩 등 검색·AI 인용을 노리는 글을 작성할 때
- 다수 홈페이지에 자동/반자동 발행할 표준 품질 콘텐츠가 필요할 때
- 병원·의료 등 규제 분야 콘텐츠(의료광고법 준수 필요)를 쓸 때

## 핵심 전제: AI는 콘텐츠를 어떻게 평가하는가
- **Legacy SEO**: 키워드 반복 빈도 중심.
- **GEO/AEO**: 콘텐츠를 문장·문단 단위로 **임베딩(벡터화)**해 주제에 대한 **의미 있는 정보의 밀도**로 평가.
  → 키워드 횟수가 아니라 "깊이·전문성·신뢰"가 노출 기준.

---

## 10대 집필 가이드라인 (반드시 준수)

1. **의미·맥락 중심 스토리** — 키워드 나열(X: "가습기 추천, 자취방 가습기 추천") 대신
   상황 설계(O: "1인 가구가 적정 습도를 유지하는 가습기 선택 기준").
2. **Q&A 구조화** — AI 유저의 자연어·질문형 검색에 맞춰 실제 질문을 **H3 소제목**으로, 아래 직답 배치.
3. **FAQ 5~7개** — 한 주제에 연관 질문 5~7개를 묶어 '종합 정보원(Authority Hub)'으로 인식시킴.
4. **역피라미드(즉답형)** — 결론·핵심을 맨 앞에. 첫 문장에 주어+목적어+서술어 완결형,
   정량 데이터('평균 3개월', '약 30%'), 확정형 종결('~입니다').
5. **외부 출처·데이터** — 주장 뒤에 검증된 통계·표·수치. 외부 채널(위키·언론) 브랜드 언급 자산화.
6. **주제 일관성(Topic Authority)** — 한 채널은 한 전문 영역만 깊게. 무관한 주제 혼재 시 정체성 스코어 하락.
7. **데이터 구조화** — 비교는 **표(table)**, 절차는 **순번 리스트(ol)**, 핵심은 불릿(ul). 흐트러진 장문 지양.
8. **고유 프레임워크** — "3단계 모델", "5가지 원칙" 등 숫자를 부여한 자체 분석틀 + 반대 관점
   ("흔히 A라 하지만, 실제 데이터는 B입니다") 1회 이상.
9. **Executive Summary + 심층 균형** — 요약(200자) → 배경(500자) → 인사이트 3개(각 300자)
   → 실행방안(400자) → ROI(200자).
10. **E-E-A-T 전방위 강화**:
    - **Experience(경험)**: "직접 3개월 써보니", "실제 집행 결과 기준" 등 인간 경험·사례.
    - **Expertise(전문성)**: 통계·출처·용어 정의. YMYL(건강·금융) 분야는 전문성 요건 엄격.
    - **Authoritativeness(권위)**: 백링크·브랜드 언급·검색량 트렌드.
    - **Trustworthiness(신뢰성)**: 성공담만이 아니라 **실패 사례와 교훈**을 넣은 글이 인용률 압도적.

> **2080 법칙**: 잘 기획된 20%의 심층 콘텐츠가 성과의 80%를 견인.
> "월 20개 단순 포스팅"보다 "분기 5개 고품질 심층 리포트".

## 권장 본문 구조 (심층 분석형 10단)

1. **즉답형 리드** `<p>` — 핵심 결론+수치 (약 200자)
2. `<h2>현황과 배경</h2>` — 시장·맥락 분석 (약 500자, 수치+출처 1회↑)
3. `<h2>핵심 인사이트 — [고유 프레임워크명]</h2>` + `<h3>` 3개 (각 약 300자)
4. `<table>` 비교표 1개 (4열↑·5행↑)
5. **데이터 막대그래프** 1개 (핵심 통계 인라인 시각화)
6. `<h2>단계별 실행 방안</h2>` — `<ol>` 5단계↑, 각 구체 액션+수치
7. `<h2>실제 사례와 교훈</h2>` — 성공 + **실패에서 얻은 교훈** 포함
8. `<h2>예상 성과와 ROI</h2>` — 구체 수치 (약 200자)
9. `<h2>자주 묻는 질문(FAQ)</h2>` — 5~7개 Q&A(각 답변 150자↑) → **FAQPage JSON-LD** 자동 생성 신호
10. `<h2>참고자료</h2>` + CTA — 인용 기관 명기(`<ul>`), 목록 밖 출처 금지.

분량 기준(참고): 심층 글은 본문 순수텍스트(태그·공백 제외) 약 3,500~4,500자.
h2 7개↑ / h3 3개↑ / table 1개↑ / ol 1개↑ / ul 2개↑.

## 출처·근거 원칙 (날조 금지)
- 통계·규정은 **공신력 기관**(정부·학회·검색 공식문서)을 근거로 본문에 기관명 명기.
- **절대 금지**: 목록에 없는 URL·논문명·보고서명·구체 통계 수치 날조.
  불확실하면 단정 대신 "일반적으로/추정".
- 논문·언론은 구체 제목·저자·날짜를 지어내지 말고 "국제 학술지(SCI) 연구에 따르면",
  "주요 일간지 보도에 따르면" 식 유형만 일반 인용.
- 전화번호 등 연락처를 지어내지 말 것. 연락 유도는 지정된 채널(예: "카카오 상담")로만.

## 의료/규제 분야 — 의료광고법 검수 (해당 시 필수)

의료·병원 콘텐츠는 **의료법 제56조** 금지 표현을 반드시 회피·자동 치환한다.
참고 구현: `lib/medical-ad-validator.js` (`validateMedicalAd`, `autoFix`).

**금지어 예시** → 안전 대체:
- 최고·최상·최초·유일·1위 → 전문적인·우수한·대표적인
- 완치·100%효과·효과보장·부작용없음 → 증상 개선·체계적 효과·부작용 최소화
- 통증없이·완전히안전 → 통증 최소화·안전 최우선
- 무료시술·경품·최저가보장 → 무료 상담·혜택·합리적 가격

절차: 초안 작성 → `validateMedicalAd()`로 검사 → 금지어 발견 시 `autoFix()`로 치환 → 재검증 통과 시 발행 가능.
비포·애프터 사진, 환자 유인성 표현 등 매체별 사전심의 대상도 함께 확인.

## 작성 절차
1. **입력 파악** — 주제/키워드, 카테고리(분야), 지역, 타깃 독자, 규제 여부.
2. (선택) **키워드 리서치** — 연관검색어·관련질문(People Also Ask)을 FAQ·소제목에 자연 반영
   (프로젝트: `lib/keyword-research.js`).
3. **초안 집필** — 위 10단 구조 + 10대 가이드라인 준수, 즉답형·정량·구조화.
4. **검수** — (의료분야면) 의료광고법 자동 검수, 깨진 태그·빈 태그·구분자 잔존 정리.
5. **구조화 데이터** — FAQ가 있으면 FAQPage, 기사면 Article JSON-LD 생성(AI Overview·리치결과 신호).
6. **출력** — 제목(숫자·순위 포함), SEO 제목, 메타설명(즉답형), 키워드 6개, 본문 HTML.

## 이것만은 반드시
1. 키워드 나열 멈추고 **검색 의도 스토리메이킹**.
2. **Q&A·FAQ** 레이아웃 전방위 배치.
3. **결론·요약을 서두에** (역피라미드).
4. 근거는 **표·수치**로, 한 채널은 **일관된 전문 영역**만.
5. **직접 경험·실패 사례·고유 프레임워크**로 E-E-A-T 집필.

## 권장 확장 (관리 콘솔 연동)
- **SEO 스킬과 짝**: 작성 직후 `seo-audit` 스킬로 발행 페이지를 점검하면 집필→검증 한 사이클.
- **다국어**: 동일 글을 언어별로 생성(프로젝트: `lib/translate.js`), hreflang 관리.
- **자동 발행 파이프라인**: 카테고리·키워드·지역만 주면 다수 사이트에 예약 발행
  (프로젝트: `api/generate-post.js`, `api/cron-daily-posts.js` 패턴).
- **이미지·인포그래픽**: 대표 이미지 프롬프트·데이터 시각화 자동 삽입.
