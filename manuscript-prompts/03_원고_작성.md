# 3단계 · 원고 작성 프롬프트 📝

> 목적: 앞에서 확정한 **주제 · 타겟 · 말투 · 목차**를 모두 합쳐 실제 원고를 써내려가는 단계예요.

---

## ✅ 복사해서 쓰는 프롬프트

```
이제 실제 원고를 써줘. 아래 조건을 모두 반영해.

[글의 조건]
- 최종 제목: [ 1단계에서 확정한 제목 ]
- 타겟(누구를 위한 글): [ 예: 하루 8시간 앉아 일하는 30대 직장인 ]
- 핵심 메시지: [ 결국 전하고 싶은 한 문장 ]
- 목차: [ 1단계에서 잡은 소제목들 — [📊 그래프] 표시 포함 ]
- 분량: [ 예: 1,500~2,000자 / 공백 포함 ]

[말투 규칙] (반드시 준수)
- 다정한 구어체 + 신뢰의 문어체를 섞는다.
  · 공감·말걸기 = 구어체("~하시죠?", "~거든요")
  · 정보·통계·치료법 = 문어체("~입니다", "~합니다")
- 독자를 아끼는 사람처럼 대하고, 겁주지 않고 안심시킨다.
- 어려운 용어는 괄호로 쉽게 풀어준다.
- 모호한 수식어 대신 구체적 수치를 쓴다.
- 한 문단 3~4줄, 과장·효능 보장·공포마케팅 금지.

[구조 규칙]
1. 도입부: 타겟의 일상 속 한 장면으로 공감하며 시작 (구어체로 말 걸기).
2. 본문: 소제목은 독자가 검색할 법한 질문형으로.
   - 핵심 답변을 소제목 바로 아래 한 문장으로 먼저 제시(즉답형).
   - 그다음 쉽게 풀어 설명.
3. 수치가 들어갈 자리에는 본문에 [📊 그래프: (어떤 데이터인지 한 줄)] 형태로 표시만 남겨둬.
   (실제 이미지는 다음 단계에서 만들 거야. 여기선 위치와 데이터만 잡아줘.)
4. 마무리: 따뜻한 응원 + 독자가 오늘 할 수 있는 행동 1가지 + 부드러운 안내(CTA).
5. 맨 끝에 'FAQ' 섹션으로 연관 질문 5개와 짧은 답변을 붙여줘.

먼저 도입부 + 첫 소제목까지만 써서 보여줘.
내가 톤을 확인하고 "이어서" 하면 끝까지 마저 써줘.
```

---

## 🧩 결과 활용법

- 완성된 원고 속 `[📊 그래프: ...]` 표시들을 모아서 4단계([`04_수치_그래프_인포그래픽.md`](./04_수치_그래프_인포그래픽.md))로 넘기면, 실제 그래프·인포그래픽 이미지가 됩니다.

## ✂️ 자주 쓰는 보조 프롬프트

```
# 더 다정하게
방금 단락이 살짝 사무적이야. 옆에서 손잡고 설명하듯 더 따뜻한 구어체로 도입부만 다시 써줘.

# 더 신뢰감 있게
이 정보 단락은 너무 가벼워. 통계와 단정형 문어체를 더해 전문가다운 신뢰감을 높여줘.

# 길이 조절
전체 톤은 유지하되 분량을 [ 1,200자 ]로 줄여줘. 핵심과 수치는 남기고 군더더기만 덜어내.
```
