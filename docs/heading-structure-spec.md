# VENOM 사이트 페이지별 H태그 구조 — 분석 & 정리

> 출처: 사장님이 정리한 페이지별 헤딩 구조를 SEO 관점에서 분석·정리한 문서.
> 원칙: **URL(문서)당 H1 1개 → H2(섹션) → H3(항목) → H4(세부)**, 순서 건너뛰기 금지(크기는 CSS로).

## 0) 한눈에 — SEO 관점 총평

| 항목 | 평가 |
|---|---|
| 헤딩 계층(H1→H2→H3→H4) | ✅ **양호** — 대부분 순서 건너뛰기 없음, 논리적 계층 |
| 페이지당 H1 1개 원칙 | ✅ 각 페이지 = 진료과목/서비스 1개 = H1 1개 |
| 키워드 배치 | ✅ H2/H3에 시술·질환 키워드 풍부(롱테일 SEO 유리) |
| 개선 필요 | ⚠️ 페이지 통합(임플란트·교정), URL 네이밍, 일부 페이지 콘텐츠 미작성 |

## 3) 실행 전 결정사항 (5)

1. **페이지 통합**: 임플란트·교정을 치과(`/dental`) H2 섹션으로 흡수할지 → 그러면 현재 배포된 `/implant`·`/orthodontics` URL 제거(진료과목 11→9). **또는** 별도 페이지 유지(각각 랭킹).
2. **URL 네이밍**: 현재 `/dental` vs 사장님 표기 `/dental-marketing`. 접미사 `-marketing` 붙일지 통일 필요.
3. **명칭 변경**: "홈페이지 제작" → "SEO 홈페이지" (나비·타이틀·본문).
4. **미작성 콘텐츠**: 정형외과 / AEO / SEO / 온라인마케팅(채널별) / 랜딩 — 헤딩·본문 작성 필요.
5. **푸터 헤딩**: 공통 푸터의 "서비스/AI마케팅/회사정보"는 페이지 H태그 계층에서 분리(또는 일관된 하위 레벨).

## 4) 권장 실행 순서
1. 결정 1~3 확정(통합·네이밍·명칭)
2. 페이지별 H1→H2→H3 마크업을 이 spec대로 구현(순서 건너뛰기 0)
3. 미작성 페이지(결정 4) 콘텐츠 작성
4. 우리 SEO 도구로 각 URL 재점검(H1 1개·계층·title·메타)
